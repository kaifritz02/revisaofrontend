Módulo 2 - Terminologia

Git é um programa VCS (sistema de controle de versão) que possui diversas funcionalidades. Umas das funções que vale ressaltar é o fácil controle de alterações feitas em um conjunto de arquivos ou códigos, além disso, expõe informações como: quem alterou, o que alterou ou como foi alterado.

Existem diversos comandos para gerenciar tais funcionalidades. Vamos apresentar os principais comandos e termos nesse método de revisão!

TERMOS:

  • Repositório: é o diretório de um projeto, localizado no nível superior de uma árvore de trabalho, no qual o Git mantém todo o histórico e os metadados de um projeto.

  • Árvore de trabalho: o conjunto de diretórios e arquivos aninhados que contém o projeto no qual está sendo feito o trabalho.

  • Commit: refere-se ao processo de armanazenar um conjunto de alterações, ou seja, de efetivar as alterações.

  • Repositório Remoto: refere-se a um projeto compartilhado, geralmente sendo armazenado no GitHub.

  • Repositório Local: diferente do repositório remoto, é onde seu trabalhado localmente, ou seja, é armazenado dentro dos arquivos internos do seu computador.
